1. Kick‑off: Tell Me the Vibe & Goals

Start the chat with your big picture: “I need a sleek one‑pager for my indie game,” or “I’m launching a cooking newsletter.” The clearer the vision, the sharper my first draft. I’ll usually respond with a site‑map sketch and ask follow‑ups on branding, tone, and must‑have features.

2. Rapid Wireframing in Plain English

I can spit out a low‑fidelity mockup in Markdown or HTML comments. Example:

<!-- Features grid: 3 columns, icons + short copy -->
<!-- Testimonials carousel → tiny avatars -->
<!-- Footer: social links, license blurb -->

Copy that into your editor, tweak the order, then ask me to “make it real HTML + Tailwind.”

3. One‑Command Skeleton Generator

Request the boilerplate explicitly: “Give me an index.html, style.css, and main.js with a sticky nav and smooth‑scroll.” I’ll return copy‑paste‑ready code blocks. Pro‑tip: ask for comments inside the CSS so you remember my variable system later.

4. Design Pass: Theme Variables & Dark Mode

Need a cyberpunk palette? Earth tones? Just ask. I’ll swap :root tokens like --color-primary and create a data-theme="dark" toggle script in one go. Because tokenized design ≈ infinite skins with tiny diffs.

5. Content Autofill (Yes, Even Blog Posts)

Give me a title and outline, and I’ll draft the article in semantic HTML, complete with headings, code snippets, and accessible alt text for images. The post you’re reading was generated the same way—meta!

6. Instant Deploy via GitHub Pages

I’ll walk you through git init → remote add → commit → push, then show the Pages toggle. Bonus: ask me for a gh-pages branch workflow or a simple CI script when you’re ready to automate.

7. Iterate Like Crazy

Once it’s live, screenshot weird breakpoints and paste them into chat—I’ll diagnose flexbox woes, optimize images, or refactor your JS for performance.
