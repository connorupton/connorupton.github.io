    How OpenAI o3 Builds (and Fixes) Your Websites

    • By Connor • April 16, 2025

    On this page

      Why write code with a language model?

      Two reasons: speed and serendipity. OpenAI o3 isn’t a replacement for human engineers, but it is the fastest pair‑programmer I’ve ever worked with. Give it a plain‑English spec—“multi‑page blog, dark mode, search, service‑worker”—and it returns scaffolded HTML, CSS variables, and even pagination logic in seconds.

      From empty folder to deployed blog

      Here’s exactly what I asked o3 when I started this site: 

      Design and develop a multi‑page blog website with a sleek and
modern UI. Requirements: Homepage, Blog list, Post pages,
pagination, dark‑mode, responsive nav…

      The model responded with a full project skeleton—file by file—plus setup instructions for GitHub Pages. All I had to do was copy the generated blocks into my repository and push.

      Automatic enhancements

      • Reading‑time & progress bar
      • Fuse.js instant search overlay
      • Prism.js syntax highlighting (see the next snippet)
      
// highlight.js or prism works out‑of‑the‑box
function greet(name) {
  console.log(`Hello, ${name}!`);
}
greet("OpenAI o3");

      Fine‑tuning on the fly

      My favorite workflow trick: paste the style.css back to o3 with a prompt like “make this theme nocturnal, use slate 900 for the background and sky 400 for primary accents.” The AI rewrites the CSS variables and even updates in‑file comments, saving me tedious color‑ picking.

      “The fastest way to learn a framework is to teach it to an AI and watch what it builds.” — someone on Twitter, probably

      But what about bugs?

      They happen—AI code is still code. The upside: you can copy the stack trace into the chat and ask o3 to reason about it. Nine times out of ten, it spots the missing await or mis‑scoped const before I do.

      Where to push next

      I’m exploring an Eleventy build step so Markdown posts generate the TOC, RSS feed, and the posts.js index automatically. For that, o3 will draft the .eleventy.js config, create a custom collection, and even write a short shell script to deploy.
